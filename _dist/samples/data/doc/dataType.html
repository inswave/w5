Data Type

w5 supports number, string, and boolean as a Data Type(3).
If specific Data Type is not designated, it maintains the same Data Type.
If specific Data Type is designated, Data will be inserted with designated Data Type(cast) when set the data or creating a grid.

Date

Data object is not a supported Primitive Data Type in w5, so in order to set the data to a grid, the data has to be cast-ed to supported Data Type.
If the data is to set with string type, the w5 pro version(paid version) can express the data in various format and original format.
Exceptionally, if 3rd Party Widget is inserted into the cell displayType of custom, the Data object can be parse to the Widget in question.

Give it a try!

The below example grid has total of 30 rows.
At col0, and col1 all the data is string type and inserted into 0 through 29.
Set col0 dataType as number.
col1 column has string dataType(default).
Sort by double-click the column header.
For your information, double-click of the header will change the sorting by Ascending, Descending, Original Order.
The sort method can be recognized by header icon.
To perform sort, set the sortable to true..

colModel[i].sortable = true;

Executing Ascending sort to col0, which dataType is number, the result is same as the original 0, 1, 2, 3 ... 9, 10, 11.
Executing Ascending sort to col1, which dataType is string, the result is 0, 1, 10, 11 ... 19, 2, 21.
