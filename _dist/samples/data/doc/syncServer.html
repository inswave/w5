Synchronize Server

Synchronize server and data with Backbone.Collection, Model method, and W5 method. Refer to sync and fetch provided by Backbone.
Refer to w5grid-rest-node sample.

Fetch

Set the URL for the communication when Backbone.Collection extend, or set it as a property of an object.

  // Set url when defining Collection.
  var List = Backbone.Collection.extend( { url: '/people' } ),
      dataCols = new List(),
      grid1 = new w5.Grid({
        ....
        collection : dataCols
      });

  // After the Collection object is created, set url property.
  var collection = Backbone.Collection.extend();
  dataCols = new collection();
  dataCols.url = '/people';

Pull data from server by calling Backbone.Collection and fetch of Modal.

  // Pull grid data by calling Backbone.Collection and fetch method.
  // reset option Re-renders the whole grid with the data obtained.
  // According to the result of synchronization, success or failure, proceed post process by setting handler.
  dataCols.fetch( { reset: true,
    success: function ( collection, response, options ) {
      // Synchronization callback process
    },
    error: function ( collection, response, options ) {
      // error process
    }
  });

  // Specific Model of Backbone.Collection proceed fetch.
  dataCols.at(0).fetch({
    success: function ( model, response, options ) {
      ....
    },
    error: function ( model, response, options ) {
      ....
    }
  });

Save

Setting save option when executing set method edits data and View and synchronizes with server at once. set method proceeds update, thereby communication with server uses HTTP PUT method. Refer to Backbone.syncfor HTTP method.
Unless otherwise directed URL through Model.urlRoot method, URL is created automatically in form of [collection.url]/[id].
Refer to Model.url and urlRootmodel for URL.

  // Edit the phone1 and phone2 column in the first row, and synchronize with server by save option at once.
  var data = { phone1: '111-111-1111', phone2: '222-222-2222' };  
  grid1.row(0).set( 'data', data, { save: true, saved: true } );

After the success of communication, saved option skips the response data parse phase and uses data that set(update) to Backbone.Model.
saved option is useful when receiving message instead of data of object that uploaded to response data. Refer to Model.parse for phase.
w5grid-rest-node example returns following response data as below:

  {
    "result": "success",
    "numberUpdated": 1
  }

pick option synchronizes specific column(property) data of Row(Backbone.Model) to server. If pick option is used, HTTP method uses PATCH.

  // Send phone1 and phone2 column data of the first row to the server.
  data = { phone1: '504-621-8927', phone2: '504-845-1427' };
  grid1.row(0).set( 'data', data, { save: true, saved: true, pick: ['phone1', 'phone2'] } );

To perform update and delete, an unique value of property or filed is required. The corresponding element can be different between client and server.
For the example of MongoDB, it uses _id as a field simliar to primary key.
In this case, use idAttribute of Backbone for mapping.

  // When defining Model, define an id field for server.
  // Below example defines _id for MongoDB.
  // When defining Collection, define Model that has been defined as model property. 
  var model = Backbone.Model.extend( { idAttribute: "_id" } ),
      List = Backbone.Collection.extend( { model: model, url: '/people' } ),

Server program example is as below:

  // Assign _id field value to a variable.
  var _id = req.params._id;
  db.collection( 'people', function ( err, collection ) {

    // Among the data received from client, _id as a key, perform update except _id field.
    req.body = _.omit( req.body, '_id');
    collection.update( { _id: ObjectID(_id) }, { $set: req.body }, function ( err, result ) {
      if (err) {
        res.json( { result: 'fail',
          message: err.message } );
      } else {
        res.json( { result: 'success',
          numberUpdated: result } );
      }
    });
  });

When performing addRow method, define save option to synchronize with server. Manage process by defining success callback handler. Perform create method, thereby uses POST HTTP method.

  data = { "first_name": "shye", "last_name": "lee", "company_name": "inswave",
           "address": "guro", "city": "seoul", "county": "korea", "state": "guro", "zip": "1234",
           "phone1": "123-456-1234", "phone2": "789-123-4567",
           "email": "shye0919@inswave.com", "web": "http://www.inswave.com" };

  // Add to the first Row and synchronize with Server.
  // Multiple(Array) add(insert) is same as Single add.
  grid1.addRow( 0, data, { save: true, success: function ( model, response, options ) {

    // success callback takes response data as the second parameter.
    // From the sample, the _id is automatically created when MongoDB performs insert.
    console.log( 'create a Row ', _.omit( response, '_id' ) );
  }} );

Destroy

If removeRow takes destroy as an option, it deletes Row and synchronizes with server at once. success callback handler can be defined as addRow. Perform delete method, thereby uses DELETE HTTP method.

  // Deletes the 2nd Row and synchronizes with server.
  grid1.removeRow( 1, { destroy: true, success: function ( model, response, options ) {
    console.log( 'removed the Row ', response.result );
  }});

Sync

Use urlCUD property and syncData method, thereby perform CUD several time, to synchronize with server at once.

When defining Collection, define urlCUD.

  var List = Backbone.Collection.extend( { model: Model, url: '/people', urlCUD: '/mixed' } );

syncData synchronizes the performed CUD to the server at once.

  grid1.addRow( 0, { first_name: 'A' } );
  grid1.cell( 2, 1 ).set( 'data', 'BP' );
  grid1.removeRow(3);
  grid1.removeRow(10);
  grid1.cell( 6, 1 ).set( 'data', 'TD' );
  grid1.addRow( 4, { first_name: 'B' } );

  // Perform server synchronization regarding Add/Edit/Delete at once.
  grid1.syncData( _.extend( options, { reset: true,
    success: function ( returnData, status, options ) {
      // perform callback 
    },
    error: function ( collection, response, options ) {
      // error process
    }
  }));

To use PATCH HTTP method, use pick option.

  • excludeCreate : Exclude created item from server synchronization.
  • excludeUpdate : Exclude updated item from server synchronization.
  • excludeDelete : Exclude deleted item from server synchronization.
  // save option synchronizes with server and skips the parse phase.
  // afterProcess option is only relevant to create method, it can be used to apply the properties, created when create is defined in the server as _id, to the Client Model.
  // The added Object to the create attribute of the - returnData - returend object  has to been defined as an Array format. 
  grid1.syncData( _.extend( options, { reset: true,
    pick: true, 
    saved: true, 
    excludeUpdate: true, 
    afterProcess: true,
    success: function ( returnData, status, options ) {
      // perform callback
    },
    error: function ( collection, response, options ) {
      // perform error
    }
  }));
