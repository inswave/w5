Samples

Selector

W5 Grid의 .get()/.set() API를 사용하기 위해서는 W5 Grid parts select method로 원하는 parts를 선택해야 합니다.
하지만 W5 Grid parts select method는 table, col, row, cell 단위로만 처리할 수 있어 다양한 조건 처리에 한계가 있습니다. 그 점을 극복하기 위해서 W5 Grid는 Selector() API를 제공합니다.

select()

select()는 W5 Grid parts select method를 대체할 수 있으므로 일반적으로 set()/get() API 이전에 W5 Grid parts를 선택합니다.

형태는 다음과 같습니다.

grid.selector( "selector strings", context, merge parts ).set(...);

Selector Strings

selector strings는 Selector의 조합으로 특정 parts를 선택합니다.

W5 Grid parts selector
CSS Type Selector에서 영감을 얻었습니다. W5 Grid parts인 table, col, row, cell을 선택할 수 있습니다.
Descendant selector
CSS Descendant Selector에서 영감을 얻었습니다. 특정 parts안에 존재하는 parts를 선택할 수 있습니다.
Property selector
CSS Atrribute Slector에서 영감을 얻었으며 W5 Grid parts에 이어 바로 []로 property를 명시하여 선택한 W5 Grid parts의 특정 Property를 가진 요소들만을 선택합니다.

예제로 상세히 알아봅시다.

grid.select("cell[data=a]").set("data", "b");
// 모든 cell들 중 data가 a"인 parts들을 선택해 data를 "b"로 set합니다.
grid.select("col[displayType=select] cell[data=0]").set("style", "background", "red");
// displayType이 select인 column들의 cell중 data가 0인 cell이 있으면 background를 red로 set합니다.

Context

selector()는 기본적으로 모든 cell들을 기본으로 합니다. 그렇기에 더 명확한 조건이 필요할 경우 두번째 인자로 Context를 지정할수 있습니다. Context는 W5 Grid parts method로 지정합니다.

grid.select( "cell[data=b]", grid1.col('col2') ) // column id가 "col2"인 셀들 중
    .set( "data", "a" )                         //data가 b인 parts들을 선택하여
    .set( "style", {                            // data를 a로 변경하고
        background : "MidnightBlue",            // background를 MidnightBlue로 지정하고,
        color : "white"                         // 폰트 색상을 white로 set합니다.
    });

Merge parts

Merge parts는 이전에 Selector()로 선택한 결과를 선택할 parts들과 merge하여 같이 처리할 수 있게끔 도와줍니다. 이전에 Selector()로 선택했던 parts들을 세번째 인자로 넘겨줍니다.

var dataA = grid1.select("cell[data=a]");  // data가 a인 셀들을 선택합니다.
grid.select("cell[data=1], grid.col('col2'), dataA").set(
    "style", {
    "background" : "MidnightBlue",
    "color" : "white"    
});
// column id가 col2인 cell들중 data가 1이거나 a인 셀들에 스타일을 적용합니다.
