Creating a W5 grid

You can create a W5 grid using simple code. Just like this!

new w5.Grid({
    el : "#grid1",       // Create a W5 grid in the DOM element of which ID is "grid1".
    collection : data,   // Insert the prepared data.
    colModel : colModel, // colModel needs to be defined in advance.
}).render();

Let's find out more!

The W5 grid creates a w5.Gird() instance using the new operator. The meta information on the grid object is defined when creating the instance, and the grid is drawn on the screen using render(). It means that the grid object can be defined differently, and can be inserted into DOM by calling render() at an intended time.
Refer to the following code snippet, if you want to create a w5.Grid() instance and draw on the screen at the same time.

var grid1 = new w5.Grid({
    el : "#grid1",
    tagName : "section",
    className : "gridStyle01",
    id : "swapID",
    option : {
        tagName : "section",
        width : "600px",
        height : "400px",
        caption : "Created W5 Grid",
        rowNum : 20,
        ...
    },
    collection : data,
    colModel : colModel,
    defaults : function () {
      return {
        email: '',
        web: '',
        ...
      };
    }
}).render();

The el property specifies the ID of the block element that becomes a target to insert the W5 grid.
The property can have a table value. In this case, the table converts to the W5Grid, using the data in the table.
For information on converting a table into W5 grid, refer to this document.

The tagName property can set the name of the first W5Grid tag name. From above code, section is to be set as the first W5Grid tag. If nothing is specified, DIV tag is used.

A class can be given to the W5 grid at the time of rendering, instead of the className API.
className given with this property will be inserted into the first tag of the W5 grid as a CSS class. As in the HTML class syntax, several classes can be given at a time, by writing a white space.

The id property specifies the ID of the element designated by el as a newly designated ID.

The collection property is the place where the data to be bound width W5Grid. The collection can be a two-dimensional array, and the JSON array type value can also be entered, which has the column ID as a key.
This collection can also be inputted by generating an instance using w5.Collection(), using a new keyword, as this collection inherits the Collection of the Backbone.

The defaults sets the initial value when the data is not available (e.g., when a new row is added). The value is passed over using function(), and the value is specified using the column ID as a key.

colModel (Column Model) is required to generated the W5 grid, and additional settings can be selected using Option. Column model and option properties are as follows.

Column Model

The colModel property defines the detailed information of columns. The detailed column properties can be set, such as width, ID, style, and display type.

Column model properties

Property Required Type Default Description
id O unique string must not empty Specify the representative key of the column in question
width X string, number 100px Specify the width of the column in question.
class X string null Input the CSS class into the cell (td) corresponding to the cell of column in question.
style X string, object null Specify the style to the cell of the column in question. Entered as a style property to cells (td).
headerLabel X string id Specify the label text of the column in question, which becomes a text node of the rendering table header (th).
hidden X boolean false Hide the column in question when rendering.
flex X number null The horizontal scroll bar is not created. The width value specified in the option is divided equally, and width is specified as much as the proportion set by flex.
sortable X boolean true Whether an event to sort (ascending, descending, sorting cancellation) the data will be bound or not, by double-clicking the label of the column in question.
dataType X string null Specify the data type of the cells corresponding to the column in question. The available type includes Number, String, and Boolean.
displayType X string "text" Specify the display type of the cells corresponding to the column in question. The available type includes text, select, checkbox, radio, link, img, button, and custom. For the custom type, the intended HTML code can be inserted using the template of the underscore.
options X object null Used when the detailed information on the item list (e.g., label and value) is needed among the displayType in question, such as checkbox, radio, and select.

Option

This section describes the property that can be set optionally, when generating the W5 grid instance.

Option properties

Property Required Type Default Description
width X stirng, number 100% The px value that will be used as grid width.
caption X string null The string that will be the grid title.
rowNum X number 10 The number of rows to be displayed for each scrolling.
vScrollDegree X number rowNum The number of rows that will be scrolled over when Scroll Area is clicked.
colOrder X array data Decide the order of columns. Put in order according to the id of colModel of the array
frozenColumn X number 0 The number of columns from the left, which will be displayed continuously even scrolled horizontally.

Live sample of W5 grid creation with a backbone collection

// Specify URL for Server and Fetch(Sync) at Backbone.Collection.
var List = Backbone.Collection.extend( { url: 'http://w5.io/dist/data/sample_499.js' } ),

    // Create Backbone.Collection Instance.
    collection = new List();

var grid1 = new w5.Grid({
    ....

    // Set the Backbon.Collection Instance to the collection property of w5.Grid
    collection : collection
});   

// Load the data by fetch which is the Backbone.Collection API.
collection.fetch({ reset: true, 
                   success: function (data) {
                     grid1.render();
                   }
});

The general fetch executes as above, but if 'Access-Control-Allow-Origin" issue occurs as below, jsonp can be used.

// Set fetch dataType option to jsonp
collection.fetch( { dataType: 'jsonp' } );

// Load data to collection at jsonp callback.
var success = function(data) {
  collection.reset( JSON.parse(data) );
  grid1.render();
}

The structure of the first Backone.Model of Collection is as follows:

collection.at(0)
