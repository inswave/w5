Concept

W5 Grid에서는 Style을 Object로 관리하고 있습니다. 그렇기에 .get()/.set() API에서도 style에 대해 .get(), .set()을 할 때에 Object로 핸들링이 가능했습니다.
기본적으로 W5 Grid Style의 정책은 적용 순서대로입니다. W5 Grid Parts select method에서 선택한 요소가 다른 요소더라도 겹치는 cell에 대해선 이후 설정된 값이 적용됩니다.
이 문서에서는 더 자세한 Rule을 설명하고자 합니다.

parts들은 각기 다른 객체

이 개념에 대해서 자세히 풀어 써 보겠습니다. 여기 5 by 5의 W5 Grid를 .set("style", ...) 상황을 표로 표현해 보겠습니다.
Parts select method로 선택한 cell들에 .set("style", ...) API를 각 기호로 매칭 시켜

  • O : grid.table().set("style", ...) 으로 table에 해당하는 cell들을 select 하여 지정한 style.
  • ▲ : grid.col(3).set("style", ...) 으로 column에 해당하는 cell들을 select 하여 지정한 style.
  • ◼ : grid.row(2).set("style", ...) 으로 row에 해당하는 cell들을 select 하여 지정한 style.
  • X︎ : grid.cell(2, 3).set("style", ...) 으로 선택한 cell에 지정한 style.

를 뜻한다고 가정해보고 표의 cell을 채워보면

col0 col1 col2 col3 col4
O O O O ▲ O
O O O O ▲ O
O ◼ O ◼ O ◼ O ▲ ◼ X O
O O O O ▲ O
O O O O ▲ O

이렇게 될 것입니다. 이것은 Style이 화면에 표현된 상황과 동일합니다.
모든 style이 적용된 cell(2,3)에 대해 .get("style")을 하게 되면 화면에 보이는 대로 O ▲ ◼ X에 대한 style Object를 얻을 수 있습니다. cell을 제외한 table, row, col은 서로 참조하지 않는 별개의 객체고 모두 제일 작은 구성단위가 cell이기 때문에 cell()로는 화면에 반영된 모든 값을 가지고 올 수 있지만 table, row, col select method로는 .set()한 값만 가지고 올 수 있습니다.

grid.table().get("style");    // O를 얻는다.
grid.cell(0, 0).get("style"); // O를 얻는다.
grid.row(2).get("style");     // ◼를 얻는다.
grid.cell(2, 1).get("style"); // O와 ◼를 얻는다.
grid.col(3).get("style");     // ▲를 얻는다.
grid.cell(0, 3).get("style"); // O, ▲를 얻는다.

Option { noncomputed : true }

그렇다면 cell()도 table, row, col과 같이 해당 객체에 .set()만 Style을 가지고 오려면 어떻게해야 할까요?
Cell의 Style에 대해 .get()할 시 마지막 인자에 Option으로 { nonecomputed: true }를 주게 되면 오로지 해당 cell의 .set("style", ...)한 값만 얻을 수 있습니다. 위의 표를 예제로 든다면
grid.cell(3, 2).get("style", { noncomputed : true })은

col3
X

와 같이 표현할 수 있습니다.

Style 적용 순위

각각의 Style 객체는 해당 객체의 요소 별로 가장 마지막에 설정한 값이 적용되고, 지정한 Style은 Parts의 DOM Node의 Style Attribute로 들어갑니다. CSS Class와 Style간의 우선순위는 보통의 HTML 문법을 따릅니다.

가장 마지막 값이 적용된다.

이전에 적용한 Style과 겹치는 cell에 대해 중복되는 Property가 있다면 가장 나중에 지정한 값이 cell에 적용이 됩니다. 그렇기 때문에 시간 차이를 두고 같은 Style Property에 대해 지정하면 layer가 겹쳐지는 듯한 화면을 볼 수 있습니다.

Style과 CSS Class

나머지 규칙은 일반 Style, CSS Class 문법을 따릅니다.
해당 DOM Element의 Style Attribute에 세팅된 Style 값이 CSS Class보다 우선순위가 높습니다.

외부(external) CSS 및 내부(Internal) CSS

Internal Style sheet가 우선순위가 더 높습니다.
