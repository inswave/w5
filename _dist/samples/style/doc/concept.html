Concept

W5 Grid manages the Style in Object. Therefore, with .get()/.set() API for the style, .get(), .set() of `Object' handling was possible.
Fundamentally, W5 Grid Style follows application order. Even W5 Grid Parts select method selects an element that overlaps other element with different style, the last style applied will be shown.
This document explains in more detail on Rule.

Each parts are different object

Let's go through in detail.
Let's express 5 X 5 W5 Grid .set("style", ...) situation in table.
Set the selected cells by Parts select method with .set("style", ...)

  • O : grid.table().set("style", ...) Set the style to the entire of the table
  • ▲ : grid.col(3).set("style", ...) Set the style to the specified column
  • ◼ : grid.row(2).set("style", ...) Set the style to the specified row
  • X︎ : grid.cell(2, 3).set("style", ...) Set the style to the specified cell
col0 col1 col2 col3 col4
O O O O ▲ O
O O O O ▲ O
O ◼ O ◼ O ◼ O ▲ ◼ X O
O O O O ▲ O
O O O O ▲ O

The table will be shown as above. Pulling the cell(2, 3), which all style has been applied, by .get("style") will return the style Object of O ▲ ◼ X.
Unlike cell, table, row, col do not refer to each other, there are separate individual objects and the smallest composition unit is a cell]. Therefore, the cell() can pull all the values but table, row, col can only pull a value set by .set() select method.

grid.table().get("style");    // Return O
grid.cell(0, 0).get("style"); // Return O
grid.row(2).get("style");     // Return ◼
grid.cell(2, 1).get("style"); // Return O and ◼
grid.col(3).get("style");     // Return ▲
grid.cell(0, 3).get("style"); // Return O and ▲

Option { noncomputed : true }

Then, how can cell() pull only the Style data set by .set() as table, row, and col does?
In the parameter of .get(), give an option parameter at the last as { nonecompute: true }. This option parameter will return the value of .set("style", ...).
grid.cell(3, 2).get("style", { noncomputed : true })

Style application order

The latest set style is applied to each Style object of each individual element. And the set style goes to the Style Attribute of DOM Node of the Parts. The order of priority between CSS Class and Style follows HTML syntax.

The latest set value is applied.

If the set style overlaps the previous cell Property, the latest set style value is applied. So that in over the time gap, if same style property has been set, the rendering will be look like the layer being overlapped.

Style and CSS Class

The rest rule follows general Style, CSS Class syntax.
The set Style to Style Attribute of DOM Element in question has higher priority than CSS Class.

External CSS and Internal CSS

Internal Style sheet has higher priority.

Selector score

In case of CSS Selector score is equal

If it's in the same file
The latest declaration value will be applied.
If it's in different css file
The latest reference file's value will be applied.
If one is the External CSS, and the other is within the HTML file (Internal CSS)
Declaration in HTML file will be applied(Internal CSS).
