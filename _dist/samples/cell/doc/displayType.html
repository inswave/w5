Display Type

The type to display the data on the screen and the following display types are currently supported:

  • text
  • select
  • checkbox
  • radio
  • link
  • img
  • button
  • custom

Template

The text and custom type can use a template. (See Custom Cell)

Event Handler

A callback handler for the change event can be specified for the select, checkbox, and radio type. Specify the handler function to the change attribute of the column meta object. (See Give it a try! 02.)

Example

The following example filters the select options of the col3 column dynamically and resets the options when the select value of the col2 column is changed:

columnObject.callback = {
    change : function( value, pos ) {
        this.cell( pos.row, 'col3' ).set( 'options', filterOptions(value) );
    }
}

Callback handler context and parameters

The this in the callback handler function refers to a grid itself.
The following parameters are passed over to the callback handler.

  • value: The value of the event-occurred cell.
  • pos[Object]: The coordinates of the event-occurred cell.

The attributes of the pos parameter (object) are as follows.

row col
coordinates of the event-occurred row coordinates of the event-occurred column

How to set

When creating a grid

General

The meta information on the column should be defined together with the data when creating a grid. Among the information, the intended widget can be expressed by specifying the displayType property.

The column information transferred at the time of grid creation is an object array. The following is an example of column object:

displayType setting according to the condition - Using an expression

The displayType of the cell in question can be specified using an expressing, depending on the different column value of the row that contains a cell.

// `obj` is an object that contains the meta information on a particular column.
obj.expression = function ( options ) {
    // Gets the value of the first column (col0).
    displayType = options.attributes['col0'];

    // Specify the `displayType` of the second column, according to the value of the first column.
    // `this` refers to a grid itself.
    // Set the `silent` option to `true` to prevent duplicated screen refreshing.
    this.cell( options.rowIndex, options.colId ).set( 'displayType', displayType, { silent: true } );

    return function (options) {
        // Returns the data of the column in question, among the entire row data (JSON - options.attributes).
        return options.attributes[options.colId];
    }
};

The parameter (options) passed over to the expression function is an object that contains the following information.

attribute value
rowIndex Row index
colId Column ID
attributes JSON object that contains the data of the row in question

The attributes JSON object contains the data of the cell corresponding to the column in question as a value. (The column ID is the name of the attribute)
The following section shows an example of the attributes JSON object:

attribute value
col0 'radio'
col1 '3'
col2 '0'
col3 '2'
col4 '0'
col5 '3'
col6 '07/08/2014'
col7 Mon Jun 30 2014

Real-Time

You can change displayType in real time, using the method of accessing the cell and column, which is provided by W5, and the set function. (See Give it a try! 01.)

// It changes the cell(0,0) displayType to 'button' from 'text'
grid1.cell(0,0).set('displayType', 'button');

// It changes the cell(0,1) displayType to 'radio' from 'select'    
grid1.cell(0,1).set('displayType', 'radio');

// It changes the 4th column displayType to 'radio' from 'select'    
grid1.col(3).set('displayType', 'select');
